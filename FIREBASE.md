# Firebase 設定（Google ログイン方式）

ログインした Google アカウントごとに、データを安全に分離して保存・同期します
（Firestore の `users/{あなたのUID}` に保存。本人のアカウントだけ読み書き可能）。
仕事用・プライベート用など、**使うアカウントを変えればデータが切り替わります。**

未設定（`src/firebaseConfig.js` が `YOUR_...` のまま）の場合は、ログインなしのローカル保存で動作します。

---

## 手順

### 1. Firebase プロジェクト & ウェブアプリ（済みならスキップ）
- `src/firebaseConfig.js` に設定値が入っていればOK。

### 2. Google ログインを有効化
1. Firebase コンソール → 左メニュー **Authentication**
2. 「始める」→ **Sign-in method** タブ
3. **Google** を選択 → **有効にする** → プロジェクトのサポートメール（自分のGmail）を選んで **保存**

### 3. 承認済みドメインの確認
- Authentication →「Settings」→ **承認済みドメイン**
- `localhost` は最初から入っています。
- **公開後はそのドメインを追加**（例：`あなたのサイト.netlify.app`）。これが無いと公開サイトでログインできません。

### 4. Firestore データベース（未作成なら作成）
- 左メニュー **Firestore Database** →「データベースの作成」→ テストモードで作成

### 5. Firestore セキュリティルール
「ルール」タブを開き、次に置き換えて **公開**：

```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // 各ユーザーは自分のドキュメントだけ読み書きできる
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}
```

### 6. 反映
- ローカル確認：`npm run dev` → ログイン画面が出る →「Google でログイン」
- 公開用：`npm run build` → `dist` を再アップ（Netlify Drop に再ドロップ）

---

## 使い方
- アプリを開くと **ログイン画面** が出ます。
- **仕事用 Google アカウントでログイン** → 仕事のデータ。
- ログアウトして **プライベート用アカウントでログイン** → プライベートのデータ。
- それぞれのデータは完全に分離され、本人のアカウントだけがアクセスできます。
- 同じアカウントで別の端末（スマホ等）から開けば、リアルタイムで同期されます。

## 以前のデータの引き継ぎ
- 旧方式（仕事/プライベート切替）で作ったデータを移したい場合：
  1. 旧バージョンで右上 **「⬇ 書き出し」** で JSON を保存
  2. 新方式でそのアカウントにログイン →「⬆ 読み込み」で取り込み

## メモ
- ログインがうまくいかない場合、多くは「Google プロバイダ未有効」か「承認済みドメイン未登録」です（手順2・3）。
- スマホでポップアップがうまく出ない場合は、リダイレクト方式に変更できます（必要なら対応します）。
